# Тесты

Smoke- и интеграционные тесты для `ml_diag`.

## Запуск

```bash
pip install pytest
pytest tests/ -v
```

## Покрытие

`test_smoke.py` содержит 21 тест, реальный корпус не требуется:

| Область | Что проверяется |
|---|---|
| Импорты | Публичные модули и подмодули `diagnosis/` импортируются без циклических зависимостей |
| Словарь меток | `PRIMARY_LABELS`, `STAGE1_LABELS`, `STAGE2_LABELS`, `to_stage1`, `to_stage2` согласованы между собой |
| Метрики | `classification_report`, `bootstrap_metric_ci`, `bootstrap_delta_ci` (инвариант «zero-when-identical») |
| Конформальный слой | `calibrate_split_conformal` даёт валидный квантиль; эмпирическое покрытие ≥ 1−α на синтетике |
| Stacking-фичеризатор | `featurize` возвращает DataFrame с документированной схемой, сохраняет индекс |
| Согласование схемы | `align_features_to_schema` удаляет лишние / добавляет недостающие / идемпотентна |
| Гибридный резолвер | Политика `agreement_or_flat` возвращает общий класс при согласии моделей |
| Evidence | `classify_evidence_notes` вызывается на минимальном входе |
| Кэш арбитратора | `_cache_key_arbitrator` детерминирован, учитывает evidence и fold_index, инвариантен к порядку ключей |
| High-level API | `diagnose(meta=..., history=df)` и `diagnose(run_dir)` → `Diagnosis` с `.save(out)` |

## Принципы тестов

- **Никакие реальные корпуса не загружаются.** Тесты работают только на синтетических данных — проверяется логика библиотеки, не полный пайплайн.
- **Никаких внешних сервисов.** Тесты не вызывают Groq или Ollama; LLM-арбитраж проверяется через `backend="template"`.
- **Детерминированность.** Все RNG используют фиксированный seed.
